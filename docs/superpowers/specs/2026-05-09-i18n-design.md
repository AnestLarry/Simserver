# i18n Internationalization for Simserver Frontend

## Overview

Add on-demand file-based internationalization (i18n) to the Svelte 5 frontend. Only the currently active language's JSON file is loaded via fetch() — no other language packs are ever downloaded. Supports English (en) and Simplified Chinese (zh-CN).

## Scope

Svelte frontend only (`svelte/panel/src/`). Go backend API response strings and `view/index.html` static page are NOT in scope.

## Architecture

### Directory Structure

```
svelte/panel/src/lib/i18n/
├── locales/
│   ├── en.json              # English translations
│   └── zh-CN.json           # Simplified Chinese translations
└── i18n.svelte.ts           # i18n store (Svelte 5 runes module)
```

Locale JSON files are also placed in `svelte/panel/public/locales/` so Vite copies them to `dist/locales/` at build time, making them fetchable at runtime.

### i18n.svelte.ts — Core Module

Uses Svelte 5 `$state` runes for reactivity:

- `_currentLocale: $state('en')` — current locale code
- `_messages: $state({})` — loaded translation key-value pairs
- `t(key, params?)` → returns translated string, falls back to key if missing; supports `{count}` style param substitution
- `setLocale(code)` → fetches `/locales/{code}.json`, updates messages, persists to localStorage
- `init()` → initializes from localStorage or navigator.language detection (zh-* → zh-CN, rest → en)
- `getCurrentLocale()` → returns current locale code

### Component Usage

```svelte
<script lang="ts">
import { t } from '$lib/i18n/i18n.svelte';
</script>

<h1>{t('app.title')}</h1>
<p>{t('chat.messages', { count })}</p>
```

### Loading Strategy

1. Page load → `init()` detects language → fetch only that language's JSON
2. User switches language → `setLocale()` fetches only the target language's JSON
3. No pre-loading, no caching of other languages — zero network requests for unused languages

### Language Switcher UI

A toggle button (EN/中) in the App.svelte top navigation bar. Position to be confirmed via Playwright visual check.

### Translation Key Organization

Keys follow a `{module}.{description}` dot-notation convention:

```
app.title              → "File Manager"
common.download        → "Download"
common.cancel          → "Cancel"
fileList.filterByName  → "Filter by name"
chat.messages          → "Messages ({count})"
upload.chooseFile      → "Choose File to Upload"
photo.noImagesFound    → "No Images Found"
```

## Testing

### Unit Tests (Vitest)

- `t()` returns correct value for existing keys
- `t()` returns the key itself for missing keys (graceful fallback)
- `t()` interpolates `{param}` placeholders
- `setLocale()` loads the correct locale file
- `init()` detects language correctly
- `init()` respects saved preference in localStorage

### E2E Verification (Python + Playwright)

After building and serving the app:
1. Launch the Go server (or serve dist/ via http.server)
2. Python Playwright script navigates to each page/view
3. Checks that no raw English key strings appear when zh-CN is active
4. Verifies language switcher works
5. Takes screenshots for manual review

## Build Pipeline

1. `npm run build` in `svelte/panel/` → outputs to `dist/`
2. Copy `dist/` contents → `view/Panel/`
3. `go build` → embeds `view/` into Go binary
4. Run binary → serves Svelte app at the configured endpoint
5. Playwright tests run against the running server
